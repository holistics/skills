# Data Modeling
## Overview
* A Data Model is an abstract representation on top of a database table/SQL query that you may manipulate without directly affecting the underlying data.
* Holistics does not automatically create Data Model for every database table. It's up to the user to transform and create the models per their needs.
* A Dataset contains multiple models and relationships.
* A Dataset CANNOT reference database tables directly. Users must create data models on top of those tables first.
